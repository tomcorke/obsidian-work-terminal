## [0.4.0] - 2026-04-20

### Highlights
- Reorganised the settings page into five clear top-level sections with dedicated sub-dialogs, making the settings page scannable at a glance.
- Unified all placeholder variables to the `$name` format and fixed `$filePath` / `$absoluteFilePath` to resolve to distinct values.
- Routed Split Task and Retry Enrichment through agent profiles so they inherit user-configured commands, flags, and login-shell wrapping.
- Added enrichment failure diagnostic logs and a last-Claude-profile deletion guardrail.

### Improvements
- Reorganised settings into five top-level sections: **General**, **Board & Columns**, **Terminal**, **Detail view**, and **Agents**. Dense groups live behind sub-dialogs (Profile Manager, Configure terminal, Configure enrichment, Configure agent actions) while small groups stay inline. (#462)
- Unified placeholder format to `$name` (camelCase, dollar prefix) across enrichment prompts, adapter prompt previews, and all settings UI. The enrichment templates now distinguish `$filePath` (vault-relative) from `$absoluteFilePath` (absolute filesystem path). (#463)
- Routed Split Task and Retry Enrichment launches through the agent-profile pipeline, inheriting command, arguments, login-shell wrapping, and profile flags. Added a **Configure agent actions** dialog exposing profile bindings. (#448)
- Moved enrichment prompt, retry prompt, profile, and timeout into a dedicated **Configure enrichment** dialog with collapsible default-prompt previews and a resolved-prompt preview helper. (#451)
- Moved the Retry Enrichment profile dropdown into the Configure Enrichment dialog so all enrichment settings are in one place. (#464)
- Consolidated the **Enable background enrichment** toggle into the Configure Enrichment dialog, removing the duplicate from the main settings list. (#461)
- Removed hardcoded `Claude (ctx)` profile-name references from settings copy; descriptions now reference the fallback chain generically. (#460)
- Added enrichment failure diagnostic logs written to `<vault>/<configDir>/plugins/work-terminal/logs/` with 7-day / 50-file retention and auto-pruning. Controlled by a new **Enrichment failure logs** toggle (default: enabled). (#449)
- Added a deletion guardrail that blocks removal of the last Claude-family profile in the Profile Manager. (#460)
- Upgraded GitHub Actions to Node 24 compatible versions. (#452)

### Fixes
- Fixed `$filePath` and `$absoluteFilePath` resolving to the same value; they now produce vault-relative and absolute paths respectively. (#465)
- Fixed non-deterministic settings page render order caused by concurrent `loadData()` calls in render helpers. (#473)
- Fixed the navigate-to-detail-view placement replacing the Work Terminal leaf instead of splitting. (#457)

### Upgrade notes
- **Breaking**: the `core.additionalAgentContext` setting has been removed. Copy your global context prompt into each relevant agent profile via the Profile Manager. Leftover values on disk are harmless. (#472)
- **Breaking**: custom enrichment prompts using `{{FILE_PATH}}` must be updated to `$filePath` or `$absoluteFilePath`. The old format no longer substitutes. (#463)

## [0.3.0] - 2026-04-17

### Highlights
- Made the board model much more flexible by decoupling task states from folders, adding state-resolution settings, deriving columns from the configured open states, and auto-cleaning empty custom columns.
- Expanded task-card customisation with pinned cards, recent-activity ordering, task icons, compact and comfortable density modes, and a setting to hide card indicators.
- Improved agent workflows with custom CLI profiles, profile-based background enrichment, per-profile login-shell wrapping, and a new `$absoluteFilePath` placeholder for context prompts.
- Strengthened developer tooling with a comprehensive user guide, cross-platform Obsidian binary detection, CI checks, and a Tier 1 CDP-based smoke-test runner.

### Improvements
- Added session filtering on the board and richer settings for badges, labels, card flags, and state ordering.
- Made background enrichment more adaptable with configurable timeouts, retries, foreground retry flows, and support for non-Claude agents.
- Improved custom CLI resolution for nvm/fnm environments, PATH augmentation, and shell-function support.
- Polished compact and comfortable card layouts with better defaults and clearer handling for pinned cards and hidden indicators.

### Fixes
- Fixed hot-reload blank-pane recovery and several compact or comfortable layout regressions, including hover shift, button gaps, and state-label edge cases.
- Fixed false active or waiting detection across hidden tabs, task switches, typed prompts, and context-launch edge cases.
- Fixed background enrichment edge cases involving task moves, timeout handling, missing-template combinations, and orphaned enrichments.
- Fixed task template and parsing inconsistencies, Windows and path-resolution edge cases, and ZWJ plus skin-tone emoji rendering for task icons.

### Upgrade notes
- Session resume and disk persistence were removed in 0.3.0. Hot reload still preserves live terminals during a plugin reload, but sessions are no longer restored across a full Obsidian restart.
- State resolution is now driven by adapter settings instead of fixed folder assumptions. Review custom columns and open-state settings if you maintain a non-default workflow.
- Compact and comfortable card modes can now intentionally hide metadata rows, and the new card-indicator setting can hide badges entirely. Review your card display settings after upgrading if metadata seems to disappear.

## [0.2.0] - 2026-04-03

### Highlights
- Replaced hard-coded agent settings with the Agent Profile Manager, including reusable profiles, colours, icons, launch summaries, and profile-aware session metadata.
- Improved terminal reliability with reopened tabs, better activity detection, dismissible resume indicators, higher scrollback, and more predictable auto-scroll behaviour.
- Made contextual prompts and background enrichment more configurable with prompt preview and suppression controls, placeholder support in extra args, timeout handling, and recovery flows.
- Expanded documentation and safety guidance with README screenshots, a source-verified process-spawning disclosure, worktree-only development guidance, and isolated Obsidian automation.

### Improvements
- Added Python 3 availability checks and clearer missing-CLI guidance before the first terminal spawn.
- Reworked the isolated-instance launcher for safer side-by-side testing and community-plugin packaging.
- Added screen-change-based Claude activity detection, background WebGL suspension, and login-shell PATH resolution for PTY spawns.
- Switched the development workflow to pnpm.

### Fixes
- Fixed tab restore, tab rename, tab alignment, and scroll-lock regressions across the terminal panel.
- Fixed Copilot and Strands context-prompt injection, resume-path edge cases, and profile settings reset issues.
- Fixed memory and resource leaks around event listeners, viewport scroll listeners, and WebGL lifecycle handling.
- Fixed blank-terminal and orphan-cleanup regressions, hash input handling, and several waiting or active false positives.

### Upgrade notes
- Agent settings moved into the Profile Manager. Review Settings after upgrading to make sure your configured profiles still reflect how you launch Claude, Copilot, and other agents.
- Development commands now use pnpm instead of npm.
- Release assets and local packaging expect `pty-wrapper.py` to ship alongside `main.js`, `manifest.json`, and `styles.css`.

## [0.1.0] - 2026-03-31

### Highlights
- First public release of Work Terminal for Obsidian.
- Shipped the core board experience with per-item shell, Claude, and contextual Claude terminal tabs.
- Added task creation, background enrichment, a guided tour, and a native Obsidian markdown detail view.
- Included CDP automation helpers, isolated test-vault support, and settings validation for external CLIs.

### Improvements
- Added session diagnostics, a public active-tab debug API, reopen-closed tabs, and persistent task or session identifiers.
- Improved task-card activity indicators, tab-restoration behaviour, and first-run setup guidance.
- Refined README and CLAUDE documentation while preparing the project for community-plugin distribution.

### Fixes
- Landed a large batch of launch-era stability fixes around WebGL teardown, contextual prompt delivery, URL opening, missing CLI guidance, and durable session recovery.
- Hardened task parsing, task IDs, waiting-state handling, and binary validation during the final pre-release stabilization pass.

### Upgrade notes
- Work Terminal is desktop-only.
- Python 3 is required for PTY-backed terminals.
- Install from the release assets or a local source checkout with `pty-wrapper.py`, `main.js`, `manifest.json`, and `styles.css` kept together in the plugin directory.
